Course Scheduling

Using greedy algorithm, where courses are assigned to available time slots based on constraints like instructor availability, room size, and student preferences.

Project Implementation

Here’s a C++ implementation of a University Course Scheduling program using a greedy algorithm. The goal is to assign courses to available time slots while respecting constraints such as instructor availability, room size, and student preferences.

Assumptions:
  1. Courses: Each course has an associated instructor and required room size.
  2. Instructors: Each instructor is available during certain time slots.
  3. Rooms: Each room has a maximum size and is available during specific times.
  4. Students: We prioritize student preferences when scheduling, but we ensure that rooms are not overbooked and instructors are available.
Approach
  1. Greedy algorithm: For each course, we attempt to assign it to the earliest available time slot that satisfies the constraints (e.g., room size, instructor availability).
  2. Input: The program takes input for courses, rooms, and instructors.
  3. Constraints:
    • Room size should be large enough for the number of students.
    • Instructors should be available in the selected time slot.
    • Time slots must not conflict with other courses.

Explanation:

  • Course Input: The program allows the user to input the number of courses, their names, the number of students, and the instructor assigned.
  • Room Input: The user inputs the rooms available with their capacities.
  • Instructor Input: The user provides the instructors and their available time slots.
  • Time Slot Input: Time slots are assigned IDs, and initially, all slots are available.
  • Greedy Scheduling: The program attempts to assign each course to the earliest available time slot and room, ensuring the room can accommodate the students, and the instructor is available.
  • Scheduling Outcome: The course schedule is printed, showing the assigned room and time slot.

business Case-6

Parking Management

Implementing an automated application and screening system, where candidates submit resumes and are evaluated based on predefined criteria.

Project Implementation

To implement a university parking management system using a hashmap (unordered_map in C++), we can design a system where each parking spot is assigned a unique identifier (ID), and the system can store and retrieve information about which cars are parked in which spots.

Key Features of the Parking Management System:

  1. Parking Spot: Each parking spot will have a unique identifier.
  2. Car Information: For each spot, we will store information about the car parked there (license plate, owner's name, etc.).
  3. User Input: The system will take user input to:
    • Park a car in a spot.
    • Check if a spot is available or occupied.
    • Retrieve details of the car parked in a spot.

      • We'll use unordered_map (hashmap) to store parking spot information where:

      • Key: Parking spot ID (integer).
      • Value: A structure holding car information (like license plate and owner's name).

Explanation:

  1. Car Structure:
    • A simple structure to hold the car's license plate and owner's name.
  2. ParkingManagementSystem Class:
    • It stores the total number of parking spots (totalSpots) and uses a hashmap (unordered_map parkingSpots) to map parking spot IDs to the car details.
    • Functions
      • parkCar: Adds a car to a specific parking spot if it's available.
      • checkSpot: Displays whether a parking spot is occupied or available.
      • removeCar: Removes a car from a parking spot.
      • showAllParkedCars: Displays all currently parked cars with their details.
  3. User Input:
    • The program prompts the user for various actions: parking a car, checking a spot, removing a car, or viewing all parked cars.
    • The user is able to enter the spot ID, car license plate, and the owner's name when parking a car.
  4. Error Handling:
    • If an invalid parking spot ID is entered or an attempt is made to park a car in an already occupied spot, the system will display an appropriate message.
